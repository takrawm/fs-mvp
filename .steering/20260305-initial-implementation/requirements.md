# 初回実装 — 要求定義

**対象:** SimpleFAM MVP 全機能の初回実装
**日付:** 2026-03-05

---

## 1. 目的

`docs/functional-design.md` および `docs/product-requirements.md` に定義された MVP 機能を、ゼロからコードとして実装する。

## 2. 実装スコープ

### 2.1 含むもの

| ID | 機能 | 概要 |
|----|------|------|
| FR-001 | シードデータ投入 | JSON → 勘定科目・実績値・ルールの登録 |
| FR-002 | 計算ルールによる将来値算出 | AST ベースの計算エンジンで予測値を算出 |
| FR-003 | CF 自動生成 | PL・BS から間接法 CF を自動生成、現預金確定 |
| FR-004 | 貸借一致検証 | 全期間の資産 = 負債 + 純資産を検証 |
| FR-005 | 財務諸表の表示 | PL / BS / CF をグリッド UI で表示 |

### 2.2 含まないもの

- イベント実行機能（LBO・M&A 等）
- CSV / Excel インポート
- ユーザー認証・権限管理
- 複数モデルの管理
- データのエクスポート

## 3. ユーザーストーリー

1. **ユーザーとして**、JSON シードデータを投入し、PL・BS の実績値をシステムに登録できる
2. **ユーザーとして**、計算ルール（成長率・割合等）を指定し、予測期間の財務数値を自動算出できる
3. **ユーザーとして**、PL・BS のデータから CF が自動生成され、現預金残高が自動計算される
4. **ユーザーとして**、全期間で貸借が一致していることを自動検証できる
5. **ユーザーとして**、PL / BS / CF をタブ切替で閲覧できる

## 4. 受け入れ条件

### 4.1 ドメインモデル

- [ ] 全値オブジェクトが不変（readonly + Object.freeze）であること
- [ ] パッケージの依存方向が `account → period → rule → computation → statement` に従うこと
- [ ] AccountHierarchy が循環参照を検出してエラーをスローすること
- [ ] 全ドメインオブジェクトに対する単体テストが存在すること

### 4.2 計算エンジン

- [ ] 7 種類のルール（ManualInput, GrowthRate, Percentage, Reference, BalanceChange, Sum, Subtract）が正しく動作すること
- [ ] DependencyGraph がトポロジカルソートで計算順序を決定すること
- [ ] 循環依存を検出してエラーとすること
- [ ] 計算結果が EvaluationCache に保存されること

### 4.3 財務諸表・CF

- [ ] PL・BS の全科目が正しく計算されること
- [ ] 間接法 CF が PL・BS から自動生成されること
- [ ] 現預金が CF から決定され、BS に反映されること
- [ ] 全期間で貸借一致が成立すること

### 4.4 サーバー・UI

- [ ] POST /api/models でシードデータを投入できること
- [ ] GET /api/models/:id/statements/pl|bs|cf で財務諸表データを取得できること
- [ ] React UI で PL / BS / CF をタブ切替で表示できること
- [ ] 実績期間と予測期間が視覚的に区別されること

## 5. 制約事項

- TypeScript で実装（strict モード）
- ドメイン層はフレームワーク非依存
- Prisma + SQLite でデータ永続化
- フロントエンドは React + Vite + react-data-grid
- バックエンドは Hono
